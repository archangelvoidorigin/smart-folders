# Always Rules — Absolute Requirements

These rules are absolute. All agents in all folders must follow them.

---

## Always Read First

- Always read `smart-folder.md` before doing any work in a folder
- Always read `settings.json` if it exists
- Always read all files in `laws/` before making changes
- Always check `.smartignore` before loading files into context

## Always Stay in Scope

- Always verify that your action falls within the folder's stated purpose
- Always check that files you create or modify belong in this folder
- Always confirm that your output format matches what the folder's connections expect

## Always Document

- Always leave the folder in a state that a future agent can understand
- Always explain your reasoning when making non-obvious decisions
- Always update `smart-folder.md` if your work changes the folder's purpose or structure
- Always write to `chronicles/` if it exists and you made significant changes

## Always Respect Hierarchy

- Always apply parent folder instructions before child-specific ones
- Always escalate conflicts to the user — do not resolve them by ignoring one side
- Always treat laws as higher priority than instructions

## Always Verify Output

- Always ensure your output is complete — no stubs, no placeholders unless explicitly requested
- Always verify that code you write is syntactically correct before finishing
- Always check that connections you define point to folders that exist

## Always Ask When Unsure

- Always ask for clarification rather than guessing
- Always flag when an instruction is ambiguous
- Always surface conflicts between instructions rather than silently choosing one

---

These requirements apply to all agents in all folders. They are the minimum standard for operating within the Smart Folder system.
